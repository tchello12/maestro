Guia de instalação e uso

Seu chief of staff em IA.

O Maestro é um sistema de agentes coordenados por um orquestrador central, rodando no Claude Code, com um “segundo cérebro” em arquivos que sobrevive entre os dias e fica mais inteligente a cada um. Este guia te leva do zip ao seu primeiro /morning.

10 agentes 11 comandos 1 brain 100% local
01

Pré-requisitos

Só uma coisa é obrigatória: o Claude Code. O chat normal do claude.ai não roda o Maestro — ele precisa de acesso aos arquivos locais, aos sub-agentes e aos comandos.

🔑

Antes de tudo, peça o acesso. No IT Connect, abra o request “Claude Enterprise (Chat + Code)” → abrir o request no ServiceNow. É o pré-requisito nº 0 — sem ele, nada roda.

  • Claude Code — app desktop (Mac/Windows), terminal ou extensão de IDE (VS Code / JetBrains). Qualquer um serve.
  • Não precisa saber programar. Você conversa em linguagem natural; o Maestro faz o resto.
  • Opcional, mas recomendado: conectores do BCG via claude.ai — Microsoft 365 (e-mail + agenda), Notion, Deckster, Knowledge Search.
💡

Se você já usa o Claude Code para outras coisas, está tudo pronto — é só abrir a pasta do Maestro.

02

Baixar o Maestro

Baixe o ZIP mais recente direto do GitHub — este link sempre traz a versão atual:

⬇  Baixar maestro.zip github.com/tchello12/maestro/releases/latest/download/maestro.zip

Clique com o botão direito no arquivo baixado → Extrair Tudo. Você terá uma pasta maestro com o CLAUDE.md dentro — coloque em algum lugar local (ex.: ~/maestro/).

  • Alternativa (git): git clone https://github.com/tchello12/maestro.git — já cria a pasta pronta e deixa você puxar updates depois com git pull.
  • Mantenha fora de drives compartilhados. O brain vai guardar material confidencial de cliente.
  • É uma cópia limpa: o sistema + templates em branco, sem nenhum dado de ninguém.

03

Abrir no Claude Code

Abra a pasta maestro/ no Claude Code (app, terminal ou IDE). Na abertura, o CLAUDE.md carrega automaticamente e o Claude vira o Maestro — você já está conversando com o orquestrador.

⚠️

Abra a pasta que contém o CLAUDE.md. Com o ZIP do link acima, ao extrair você tem uma pasta maestro com o CLAUDE.md dentro — abra exatamente essa (File ▸ Open Folder). Se você baixar pelo botão "Code ▸ Download ZIP" do GitHub, pode vir pasta dentro de pasta (maestro-main/maestro-main/) — aí entre até achar o CLAUDE.md. Regra: ao abrir, você tem que ver o CLAUDE.md e a pasta .claude/ na raiz.

✅

Como saber que abriu o nível certo: a primeira resposta vem como o Maestro (e não "sou o Claude Code, ajudo com desenvolvimento de software"), e ao digitar / o autocomplete mostra /setup, /morning etc. Se ele se apresentar como assistente de dev ou só aparecerem comandos genéricos, você abriu o nível errado — desça uma pasta.

O atalho: setup guiado built-in

Você não precisa seguir os passos abaixo na mão. Na primeira vez, é só digitar:

/setup

O Maestro detecta que é o primeiro uso, faz um diagnóstico do que falta (identidade, perfil, integrações, objetivos, caso) e te conduz passo a passo — uma pergunta de cada vez, aceitando brain-dump e estruturando os arquivos por você. É resumível: pare e rode /setup de novo quando quiser; ele só faz o que ainda falta.

✨

Os passos 04 a 08 a seguir são exatamente o que o /setup cobre. Leia se quiser entender o que acontece — ou pule direto pro /setup e deixe o Maestro guiar.

04

Personalizar a identidade

O template vem com placeholders. A forma mais fácil de trocar tudo de uma vez é simplesmente pedir ao Maestro:

"me configura — sou o(a) [Nome Completo], [email@bcg.com], [seu nível] no BCG [escritório]"

Ele substitui os marcadores {{OWNER}}, {{SEU_NOME_COMPLETO}} e {{SEU_EMAIL}} em todo o sistema. É também no CLAUDE.md que se ajusta o enquadramento caso você não seja do BCG.

05

Preencher o perfil

O passo de maior alavancagem — o Maestro lê o perfil toda sessão. Você não precisa editar à mão: faça um brain-dump e ele estrutura os arquivos.

  • working-preferences.md — nível, horários de pico, como você gosta dos outputs, o que nunca explicar pra você, tempo protegido. Comece por aqui.
  • bio.md — a pessoa por trás do crachá: trajetória, vida além do BCG, o que te move.
  • case-history.md — seus casos, skills, spikes e gaps. Alimenta prep de CDC e staffing.
🧠

Quanto mais fiel o brain, mais afiado o Maestro. Ele é exatamente tão inteligente quanto o brain está atualizado.

06

Conectar integrações

Opcional, mas é o que destrava o brief da manhã e o sync de tarefas.

  • Microsoft 365 (Outlook + agenda) — alimenta o /morning. O Claude pede autenticação na primeira vez que o briefing-analyst rodar.
  • Notion — se você mantém tarefas lá, preencha brain/tasks/notion-sync.md no primeiro uso (o Maestro sonda suas bases). Se não usa, ignore.
  • Conectores BCG — Deckster (slides/ThinkCell), Knowledge Search, Transcript Library, CapIQ — conforme disponíveis na sua conta.
07

Semear os objetivos de desenvolvimento

Rode /feedback com seu feedback de projeto mais recente e sua última avaliação formal. Isso popula development/objectives.md — a base do nudge diário e do retro de sexta.

🎯

É o que transforma o Maestro de “assistente de tarefas” em parceiro de carreira: ele conecta o trabalho do dia aos seus objetivos e, na hora do CDC, compila a evidência que você já registrou.

08

Onboardar seu caso atual

Aponte o Maestro para o deck de proposta do seu caso:

/newcase [caminho para o deck de proposta .pdf ou .pptx]

O case-onboarder minera o deck, rascunha os arquivos de cliente e projeto, e te pergunta só o que o deck não responde. O melhor momento pra capturar contexto é o início do caso — faça uma vez, bem feito.

09

Ir ao ar

No próximo dia útil, abra um chat novo e digite:

/morning

O Maestro lê o brain, puxa sua agenda e e-mails, e te entrega um brief priorizado: o formato do dia, top 3 prioridades, watch-outs, um nudge de desenvolvimento e o primeiro movimento. Você está no ar.

10

O ritmo diário

O hábito que faz o sistema funcionar — o valor compõe com a consistência.

Manhã
Chat novo → /morning. Lê o brain, puxa agenda + e-mail, dá o brief priorizado e abre o log do dia.
Ao longo do dia
Só conversa. Conte o que aconteceu (“reunião com o CFO, preocupado com prazo”) → ele captura. Peça builds (slides, análises) → ele delega.
Fim do dia
/eod. Finaliza o log, atualiza o backlog, recap em 3 linhas + a primeira prioridade de amanhã.
Sexta
/retro. Revisa a semana contra seus objetivos, com evidências dos logs. Fecha uma intenção pra próxima semana.
⭐

Regra de ouro: uma conversa por dia. O chat é descartável — o brain é permanente. /eod e /capture garantem que nada durável morra com o chat.

11

Comandos & agentes

Os comandos são atalhos pros rituais — no resto, é só conversar normalmente.

11 comandos

Brief completo: brain + agenda + e-mail + priorização + nudge
/eod
Fecha o dia: log, backlog, recap + amanhã
/capture
Arquiva tarefa / fato / decisão no lugar certo do brain
/task
Operações na to-do do Notion: add / done / mover / listar
/schedule

Te briefa sobre um cliente, ou atualiza o arquivo dele
/newcase
Onboarda um caso a partir do deck de proposta
/retro
Retro semanal de desenvolvimento (sextas)
/feedback
Arquiva feedback formal e atualiza seus objetivos
/meeting-to-work-items
Transcrição → decisões + tarefas (owner/prazo) + follow-ups, arquivados
/setup
Onboarding guiado de primeiro uso — preenche o brain e conecta tudo

10 agentes especialistas

Você nunca os chama direto — o Maestro delega conforme o que você pede.

briefing-analyst
Lê Outlook + agenda + transcrições; devolve um digest
case-onboarder
Minera o deck de proposta → arquivos do brain
client-keeper
Mantém o conhecimento dos clientes (pessoas, política, decisões)
work-logger
Logs diários, backlog e status de projeto
work-advisor
Priorização: o que fazer primeiro e o que NÃO fazer
slide-builder
Storylines BCG + gráficos ThinkCell via Deckster
analyst
Python + Excel: modelos, dados, exhibits
support-coach
O lado humano: descompressão, prep de conversas difíceis
challenger
Pressure-test (lógica · risco · audiência) antes de subir pra PL/MDP ou cliente
artifact-reviewer
Confere o entregável vs. o brief antes de mandar ao cliente
🛠️

Builder Mode (não é comando — é só falar): diga “builder mode” e o Maestro troca de chapéu pra construir outros agentes e skills. O sistema se auto-expande — mas mudanças em agentes/comandos só valem após reiniciar a sessão.

12

Boas práticas & confidencialidade

  • Chat novo todo dia. Conversas longas degradam; o brain não. /eod e fecha; /morning num chat novo.
  • Capture sem dó. O sistema é tão inteligente quanto o brain está atual. Na dúvida, /capture.
  • Trate os primeiros runs como piloto. Errou? “builder mode” e corrija — o fix é permanente.
  • Confidencialidade. O brain tem dado de cliente: pasta local, política de dados do BCG, nunca cole conteúdo do brain em ferramentas não-sancionadas. Prefira um gerenciador de senhas a texto plano.
